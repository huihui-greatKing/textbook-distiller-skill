# AGENTS.md

本文件用于约束之后 Codex 在本项目中的工作方式。项目定位是学生课程实践，目标是辅助理解教材、整理知识框架、规范解题过程和形成课程作业材料。

## 1. 工作原则

- 不要凭空编造教材内容；
- 必须基于用户提供的教材 PDF、截图、PPT 或笔记；
- 当用户明确说“不要放教材内容”时，只能优化 Skill、模板、规则和检查表；
- 所有教材内容处理必须先经过资料清单和可读性检查；
- 如果 PDF 是扫描版，应先进行 OCR 或图像识别；
- 不得在没有 OCR 或文本依据的情况下总结教材；
- 章节蒸馏前必须确认用户授权范围；如果用户明确要求整本 OCR、全书蒸馏或直接继续处理，可以按授权范围连续执行；
- 对识别不清楚的公式、图表、页码、表格必须标注“待人工复核”；
- 所有不确定内容必须写入待复核项；
- 不要大段照抄教材原文；
- 要提炼教材的知识结构、解题方法和表达方式；
- 处理教材前先建立资料清单和章节结构；
- 逐章处理时先列框架，再补充概念、公式、图表、题型和方法；
- 逐章蒸馏时应先建立候选知识点、候选题型和候选方法，再筛选进入章节 Skill；
- 未进入章节 Skill 的重要候选项应记录淘汰、合并或待复核原因；
- 每个章节 Skill 必须有明确调用条件、不应调用的情况和测试提示词；
- 跨章节关联应单独记录，方便综合作业 Skill 判断题目归属；
- 不确定内容不能写成确定结论；
- 不要把阶段限制写死；以用户当前授权为准，允许在明确授权后生成第 1 章到第 10 章的章节总结、章节 Skill、测试和报告。

## 2. 输出风格

- 使用中文；
- 保持学生课程作业风格；
- 输出语气保持学生课程项目风格；
- 不要商业化；
- 不要夸大 AI 作用；
- 不要写成“AI 完全替代学习”；
- 要写成“辅助理解、辅助复习、辅助规范作业书写”；
- 表述要清楚、朴素、可检查；
- 涉及作业答案时要保留必要过程，不只给结论。

## 3. 文件规范

- 所有章节 Skill 放入 `skills/chapter-xx-xxx/SKILL.md`；
- 所有测试题放入 `tests/test-prompts.md`；
- 所有评价标准放入 `tests/evaluation.md`；
- 所有示例放入 `examples/`；
- 教材原始资料放入 `knowledge-base/raw/`；
- OCR 记录放入 `knowledge-base/evidence/ocr-notes/`；
- 图示整理记录放入 `knowledge-base/evidence/figure-notes/`；
- 公式和表格复核记录放入 `knowledge-base/evidence/formula-table-notes/`；
- 候选与淘汰记录放入 `knowledge-base/evidence/candidate-notes/`；
- 结构化概念放入 `knowledge-base/normalized/concepts/`；
- 题型整理放入 `knowledge-base/normalized/problem-types/`；
- 跨章关联索引放入 `knowledge-base/normalized/cross-links.md`；
- 不确定内容统一写入 `evidence/` 或待复核列表。

## 4. 完成标准

- 有清晰目录结构；
- 有教材蒸馏流程；
- 有章节 Skill 模板；
- 有测试验证方法；
- 有候选筛选和淘汰记录；
- 有章节调用边界和跨章关联记录；
- 有课程报告生成方案；
- 能说明每个输出文件的来源；
- 能区分教材原始资料、识别结果、整理结果和待复核内容；
- 能保证章节 Skill 可单独调用；
- 能通过测试题检查 Skill 的调用效果。
