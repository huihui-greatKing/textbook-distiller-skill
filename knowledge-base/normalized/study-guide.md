# 《计算机组成与设计》复习式蒸馏总览

本文件是面向复习和作业的总览，不是模板清单。它把第 1 章到第 10 章整理成“学什么、怎么答题、哪里容易错”的形式。

## 第1章 绪论

本章主线：建立计算机系统的整体认识，理解硬件、软件、指令、程序和性能指标之间的关系。

核心知识点：
- 计算机系统：由硬件系统和软件系统共同组成，硬件负责执行，软件负责组织任务和提供使用方式。
- 存储程序思想：程序和数据都放在存储器中，CPU 按指令顺序取出并执行，这是理解后续 CPU 与存储系统的基础。
- 硬件系统：包括运算器、控制器、存储器、输入设备和输出设备，常用于说明计算机组成框架。
- 软件系统：包括系统软件和应用软件，用来管理硬件资源并完成用户任务。
- 性能指标：常见指标包括主频、字长、存储容量、响应时间、吞吐率等，答题时要区分指标含义。

作业方法：
- 概念解释题：先给定义，再说明作用，最后联系后续章节。
- 组成结构题：按五大部件写，并说明各部件之间通过数据和控制信息配合。
- 性能指标题：不要只说越大越好，要说明指标适用场景。

复习提醒：绪论类作业要写清楚“系统层次”和“部件作用”，少用空泛表述，多用结构化分点。

## 第2章 计算机中的数据表示

本章主线：回答数据在计算机中如何编码、如何表示正负和小数、如何检查传输或存储中的错误。

核心知识点：
- 机器数：计算机内部实际存放的二进制编码形式。
- 真值：机器数所代表的实际数值，是人理解的数学值。
- 原码：最高位表示符号，其余位表示绝对值，直观但加减运算不方便。
- 补码：负数通过模运算思想表示，可把减法转化为加法，是计算机常用的有符号数表示。
- 反码：负数除符号位外各位取反，常作为理解补码的过渡。
- 定点数：小数点位置固定，适合整数或固定小数范围的表示。

作业方法：
- 数制转换题：写出转换依据，保留中间步骤，避免只给结果。
- 机器数表示题：先判断位数和符号，再写编码过程。
- 编码比较题：从表示方式、范围、运算方便性比较。
- 校验码题：说明校验规则、生成过程和可发现的错误类型。

复习提醒：数据表示题最容易丢步骤。作业中应写清位数、符号位、转换过程、最终编码和是否需要复核公式符号。

## 第3章 运算方法与运算器

本章主线：理解计算机如何在二进制编码基础上完成加减乘除、浮点运算和 ALU 运算。

核心知识点：
- 定点运算：在固定小数点位置下进行加、减、乘、除。
- 补码加减法：把减法转化为加法处理，硬件实现更统一。
- 溢出：运算结果超出给定位数可表示范围，需通过符号或进位关系判断。
- 移位运算：通过左移或右移实现乘除 2 的幂或规格化处理。
- 浮点运算：通常包括对阶、尾数运算、规格化、舍入和溢出判断。
- ALU：算术逻辑单元，执行加减、逻辑、移位和状态标志生成。

作业方法：
- 补码运算题：写补码、列加法、判断溢出、写真值结果。
- 浮点运算题：按对阶、运算、规格化、舍入写步骤。
- ALU 结构题：从算术、逻辑、移位、标志位四点回答。

复习提醒：运算题必须保留中间编码和溢出判断，不能只给十进制结果。

## 第4章 指令系统与汇编语言

本章主线：理解机器指令如何表示操作、操作数和地址，以及汇编语言如何对应机器执行过程。

核心知识点：
- 指令系统：计算机能够识别和执行的全部指令集合。
- 操作码：指出指令要完成的操作。
- 地址码：指出操作数或操作数地址。
- 寻址方式：确定操作数位置或地址的方法。
- 指令格式：操作码、地址码等字段的组织形式。
- 汇编语言：用助记符表示机器指令，便于程序员书写。

作业方法：
- 指令格式题：画出字段或分点说明。
- 寻址方式题：先判断寻址方式，再列地址计算过程。
- 汇编分析题：逐条说明数据流向和结果。

复习提醒：指令题要避免只翻译助记符，应写出地址形成过程和指令执行效果。

## 第5章 处理器(CPU)设计

本章主线：理解 CPU 如何通过数据通路和控制器完成指令执行。

核心知识点：
- CPU：由运算器、控制器、寄存器等组成，负责取指、译码和执行。
- 数据通路：数据在寄存器、ALU、存储器和总线之间流动的路径。
- 控制信号：控制各部件在正确时序完成操作的信号。
- 硬布线控制：用组合逻辑和时序逻辑产生控制信号，速度快但修改困难。
- 微程序控制：用微指令组织控制过程，结构清晰且便于修改。
- 指令周期：一条指令从取指到执行完成所经历的过程。

作业方法：
- 指令执行流程题：按阶段列出寄存器传送和控制信号。
- 数据通路分析题：结合图写出数据流向。
- 控制方式比较题：从实现方式、速度、可修改性比较。

复习提醒：CPU 题要把“数据通路”和“控制信号”一起写，否则答案容易空泛。

## 第6章 流水线技术

本章主线：理解流水线如何通过重叠执行提高吞吐率，以及相关冲突如何影响性能。

核心知识点：
- 流水线：把指令执行划分为多个阶段，使多条指令在不同阶段重叠执行。
- 吞吐率：单位时间内完成的任务或指令数量。
- 加速比：非流水执行时间与流水执行时间之比。
- 效率：流水线各段实际利用程度。
- 结构相关：多条指令争用同一硬件资源产生的冲突。
- 数据相关：后一条指令需要前一条指令结果导致的等待。

作业方法：
- 性能计算题：先写公式含义，再代入数据。
- 相关分析题：说明冲突原因和解决方法。
- 时空图题：标清周期、阶段和停顿。

复习提醒：流水线题要画清阶段和周期，公式必须结合题目给定段数和任务数。

## 第7章 存储系统

本章主线：理解存储层次如何在速度、容量和成本之间折中，并掌握 Cache 和虚拟存储器的基本方法。

核心知识点：
- 存储层次：由寄存器、Cache、主存、外存等组成，越靠近 CPU 通常越快、容量越小、成本越高。
- RAM：随机读写存储器，主存常用类型。
- SRAM：静态 RAM，速度快、成本高，常用于 Cache。
- DRAM：动态 RAM，容量大、成本低，需要刷新，常用于主存。
- ROM：只读或以读为主的非易失存储器。
- Cache：位于 CPU 和主存之间，用局部性原理提高平均访存速度。

作业方法：
- 存储器分类题：从是否易失、速度、成本、用途比较。
- Cache 分析题：先判断映射方式，再分析标记、索引和块内地址。
- 虚拟存储题：拆分虚拟地址，查页表，再得到物理地址。

复习提醒：存储系统题要把“为什么这样设计”和“怎么算”结合起来，尤其 Cache 题要写映射和替换依据。

## 第8章 输入/输出系统

本章主线：理解外设如何通过接口、程序查询、中断和 DMA 与 CPU/主存交换数据。

核心知识点：
- I/O 接口：连接主机和外设，完成数据缓冲、状态控制和信号转换。
- 端口：CPU 与 I/O 设备交换信息的寄存器或地址。
- 程序查询：CPU 主动轮询设备状态，简单但效率低。
- 中断：设备主动请求 CPU 服务，提高 CPU 利用率。
- DMA：外设与主存直接传输数据，减少 CPU 参与。
- 总线：连接 CPU、主存和外设的数据、地址、控制通道。

作业方法：
- I/O 方式比较题：列出优缺点和适用场景。
- 中断过程题：按流程写，不要只说“打断 CPU”。
- DMA 分析题：写清 CPU、DMA 控制器、主存和外设分工。

复习提醒：I/O 题要围绕“谁控制传输、CPU 是否等待、效率如何”来组织答案。

## 第9章 多机系统

本章主线：理解多个处理单元如何通过并行处理、共享存储或消息传递协同工作。

核心知识点：
- 多机系统：由多个处理单元协同完成任务的系统。
- 并行处理：多个操作同时或重叠执行以提高处理能力。
- 多处理机：多个处理器通常共享存储资源。
- 多计算机：多个计算机节点通过网络互连，常通过消息传递协作。
- 互连网络：连接处理器、存储器或节点的通信结构。
- 加速比：并行系统相对串行系统的性能提升程度。

作业方法：
- 结构比较题：从存储共享、通信方式、扩展性比较。
- 性能分析题：说明理想加速和实际开销。
- 互连网络题：写清连接对象和数据传输方式。

复习提醒：多机系统题不要只写“多个 CPU”，要说明存储组织、通信方式和并行开销。

## 第10章 基于 EDA 的计算机设计

本章主线：理解如何用 EDA 方法和硬件描述语言完成计算机部件建模、仿真、综合和验证。

核心知识点：
- EDA：电子设计自动化，用软件工具辅助硬件设计、仿真、综合和实现。
- 硬件描述语言：用文本描述硬件结构或行为，如 Verilog/VHDL。
- 模块化设计：把复杂硬件分解为接口清晰的模块。
- 仿真：在实现前检查设计行为是否符合预期。
- 综合：把硬件描述转换为门级或可实现结构。
- 验证：检查设计结果是否满足功能和时序要求。

作业方法：
- 流程说明题：按流程分点，写每一步目的。
- 模块分析题：说明输入、输出、内部逻辑和连接关系。
- 实验报告题：写清设计目标、方法、结果和不足。

复习提醒：EDA 题要避免只贴代码，应解释代码对应的硬件结构和验证结果。

4. 用第 7-8 章理解存储和输入输出；
5. 用第 9-10 章理解多机系统和 EDA 设计；
6. 做作业时先判断章节，再调用对应章节 Skill。
