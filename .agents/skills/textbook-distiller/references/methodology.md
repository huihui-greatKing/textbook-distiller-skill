# 教材蒸馏方法论

本文件说明 `textbook-distiller` 的核心方法。只有在需要规划或执行教材蒸馏时读取。

## 1. 方法来源与取舍

教材蒸馏不应停留在“把书总结成几段话”，而应把教材变成可调用、可检查、可迭代的学习工具。本项目采用的通用方法包括：

- 证据驱动整理：类似资料库和 RAG 流程，所有结论都能回到原始资料；
- 学习目标设计：围绕概念理解、方法掌握、题型迁移和作业表达设计输出；
- Rubric 评价：先定义好答案质量标准，再生成和测试 Skill；
- 分阶段流水线：每个阶段只处理一个目标，减少混杂输出；
- 人工复核门禁：把 OCR、页码、公式、表格、图示的不确定性显式保留；
- 压力测试：用边界题和诱导题检查 Skill 是否越界。

## 2. 运行模式

`textbook-distiller` 支持两种运行模式：

| 模式 | 使用条件 | 执行方式 |
| --- | --- | --- |
| 阶段确认模式 | 用户只要求某一阶段、资料不完整、OCR 工具未验证、或需要人工确认 | 每完成一个阶段就输出结果和下一步建议 |
| 全流程授权模式 | 用户明确要求整本 OCR、全书蒸馏、总结具体章节、生成全部章节 Skill 或“直接继续做” | 连续推进资料登记、OCR、章节切分、章节总结、章节 Skill、测试和报告；遇到严重阻塞才停止 |

全流程授权模式不是放松质量要求，而是减少不必要停顿。局部不确定内容应写入待复核项，严重阻塞才停止。

## 3. 证据记录

每条重要内容尽量记录：

| 字段 | 说明 |
| --- | --- |
| 来源文件 | PDF、截图、PPT 或笔记文件名 |
| 教材页码 | 原书页码，未知则标注待复核 |
| PDF 页序 | PDF 中的页序，和教材页码分开记录 |
| 内容类型 | 概念、公式、图表、例题、题型、方法 |
| 置信状态 | 已确认、OCR 待复核、人工待复核 |
| 备注 | 不确定原因或处理说明 |

## 4. OCR 与章节文本包

扫描版教材应先形成 OCR 证据层，再进行章节蒸馏。推荐输出：

- `knowledge-base/evidence/ocr-notes/fulltext/manifest.json`：整本 OCR 运行元数据；
- `knowledge-base/evidence/ocr-notes/fulltext/pages/page-xxx.md`：逐页 OCR 文本；
- `knowledge-base/evidence/ocr-notes/fulltext/jsonl/page-xxx.jsonl`：逐行 OCR、置信度和坐标；
- `knowledge-base/normalized/chapter-texts/chapter-xx.md`：按章节切分的 OCR 文本包；
- `knowledge-base/normalized/chapter-summaries/chapter-xx-summary.md`：章节总结和待复核项。

OCR 文本包是工作材料，不等于最终总结。章节 Skill 应从 OCR 文本中提炼结构、方法和题型，不应大段复制 OCR 原文。

## 5. 候选筛选

逐章蒸馏时先建立候选池，再筛选进入章节 Skill。

候选类型：

- 核心概念；
- 重要定义；
- 公式和规则；
- 图表；
- 例题；
- 常见题型；
- 解题方法；
- 易错点；
- 教材表达方式。

处理结果：

- 纳入：进入章节 Skill；
- 合并：和相近内容合并；
- 淘汰：内容太细、重复、对作业价值较低；
- 待复核：证据不清楚或 OCR 质量不足。

## 6. 原子化章节 Skill

章节 Skill 不是章节笔记，而是一个可调用工具。它必须回答：

- 什么时候调用；
- 什么时候不调用；
- 需要什么输入；
- 输出什么结构；
- 如何保留教材依据；
- 如何处理不确定内容；
- 如何自检；
- 如何测试。

## 7. 跨章关联

跨章关联用于综合作业 Skill 的路由和判断。

每章记录：

- 前置知识；
- 后续知识；
- 容易混淆的相邻章节；
- 可能跨章的题型；
- 综合作业调用提示。

输出位置：`knowledge-base/normalized/cross-links.md`。

## 8. 增量运行

每次只推进一个阶段或一个章节。新增内容时保留已有资料，不覆盖人工复核记录，不把未确认内容改成确定结论。

如果用户已经授权全流程运行，则把“增量”理解为可恢复的流水线：每个阶段都落盘生成中间结果，后续失败时可以从最近的中间结果继续，而不是重新处理全部页面。
