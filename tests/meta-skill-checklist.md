# textbook-distiller 元技能检查表

本文件用于检查 `skills/textbook-distiller/SKILL.md` 是否已经具备稳定、清晰、可执行的元技能结构。检查对象是元 Skill 本身，不是教材内容。

| 检查类别 | 检查问题 | 合格标准 | 检查结果 | 备注 |
| --- | --- | --- | --- | --- |
| 结构完整性检查 | 是否包含元信息、定位、边界、流程、规则、模板和示例指令 | 模块完整，标题清晰，能指导后续执行 | 待检查 |  |
| 输入检查是否完整 | 是否要求检查教材名称、文件、范围、输出目标、OCR、图表、作业模板和报告需求 | 信息不完整时先输出缺失信息清单 | 待检查 |  |
| 阶段流程是否清晰 | 是否将任务拆成任务确认、资料登记、可读性检查、目录计划、逐章计划、章节 Skill、综合 Skill、测试、报告 | 每个阶段有目标、规则和输出 | 待检查 |  |
| 强制停止规则是否明确 | 是否说明遇到不可读 PDF、页码异常、OCR 不清楚、用户限制等情况必须停止 | 停止时说明完成内容、停止原因、需要确认内容和下一步建议 | 待检查 |  |
| 反幻觉规则是否明确 | 是否禁止无依据编造、用常识替代教材、把不确定内容写成结论 | 不确定内容统一进入待人工复核项 | 待检查 |  |
| 输出格式是否统一 | 是否规定本轮完成内容、新增或修改文件、关键结论、待复核项、下一步建议和是否继续 | 每轮输出格式稳定，便于检查 | 待检查 |  |
| 是否适合扫描版教材 | 是否包含 PDF 可读性检查、OCR 规划、图表公式复核和页码对应关系 | 能处理扫描版或图片版教材 | 待检查 |  |
| 是否能支持后续章节 Skill 生成 | 是否提供章节 Skill 标准模板和生成条件 | 只有用户确认后才生成章节 Skill | 待检查 |  |
| 是否能支持课程报告生成 | 是否说明课程报告内容、语气和边界 | 报告保持学生课程实践风格 | 待检查 |  |
| 是否避免直接处理教材内容 | 是否明确用户要求只优化 Skill 时不得读取教材、OCR、识别目录或生成章节 Skill | 能防止越界进入教材内容处理 | 待检查 |  |
| 是否支持不同教材复用 | 是否避免绑定某一本教材、某一门课程或固定章节名 | 换教材时只替换资料、范围和目标输出 | 待检查 |  |
| 是否有证据追踪机制 | 是否要求知识点、公式、图表、例题和方法都能追溯来源 | 每个结论都有资料依据或待复核标注 | 待检查 |  |
| 是否有候选筛选机制 | 是否先建立候选知识点、题型和方法，再筛选进入章节 Skill | 有候选、纳入、合并、淘汰、待复核记录 | 待检查 |  |
| 是否有调用边界 | 是否要求章节 Skill 写明调用条件和不应调用的情况 | 能避免题目误路由或过度调用 | 待检查 |  |
| 是否有跨章关联机制 | 是否要求记录前置知识、后续知识和相邻章节混淆点 | 能支持综合作业 Skill 判断题目归属 | 待检查 |  |
| 是否有压力测试设计 | 是否包含正常题、跨章题、边界题、证据不足题和诱导题测试 | 能检查 Skill 是否越界、编造或只给结论 | 待检查 |  |
| Codex Skill 结构是否有效 | 是否能通过 `quick_validate.py` 校验 | 输出 `Skill is valid!` | 待检查 |  |
| UI 元数据是否存在 | 是否包含 `skills/textbook-distiller/agents/openai.yaml` | 有显示名、简短说明和默认调用提示 | 待检查 |  |
| Python 依赖是否记录 | 是否包含 `requirements.txt` | 基础依赖可一键安装，OCR 重型依赖后续按需补充 | 待检查 |  |
| 渐进加载结构是否清晰 | 是否将长模板和评测规则拆入 `references/` | `SKILL.md` 保留核心流程，reference 按需读取 | 待检查 |  |
| 方法论是否完整 | 是否包含证据追踪、候选筛选、原子化章节 Skill、跨章关联和增量运行 | 能指导不同教材复用 | 待检查 |  |
| 模板是否集中管理 | 是否把输入检查表、输出格式、文件命名和章节 Skill 模板集中到 `references/templates.md` | 后续改模板时不需要改多处 | 待检查 |  |
| 评测门禁是否集中管理 | 是否把强制停止、反幻觉、测试矩阵和评分 rubric 放到 `references/evaluation.md` | 能稳定检查章节 Skill 质量 | 待检查 |  |
