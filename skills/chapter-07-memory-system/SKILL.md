# 第7章 存储系统 Skill

## 1. Skill 目标

把第7章“存储系统”蒸馏成学习型 Skill。调用时要讲清概念、方法、题型和作业过程，不能只给章节概要或一句结论。

## 2. 本章学习目标

- 比较 SRAM、DRAM、ROM 等存储器
- 解释存储层次和局部性原理
- 分析 Cache 映射、命中率和平均访存时间
- 完成虚拟地址到物理地址的转换思路

## 3. 本章知识地图

- 存储层次：寄存器、Cache、主存、外存
- 主存：RAM、ROM、SRAM、DRAM
- Cache：映射、替换、写策略、命中率
- 虚拟存储：页、页表、地址转换

## 4. 核心概念精讲

| 概念 | 是什么 | 为什么引入 | 怎么用 | 怎么考 | 易混点 |
| --- | --- | --- | --- | --- | --- |
| 存储层次 | 由速度、容量和成本不同的存储部件构成的层级。 | 单一存储器无法同时满足高速、大容量、低成本。 | 回答为什么需要 Cache 和外存时使用。 | 常考层次结构和特点。 | 不要只按容量排序，忽略速度和成本。 |
| SRAM | 不需刷新的静态随机存储器。 | 速度快但成本高。 | 常用于 Cache。 | 常考与 DRAM 比较。 | 不要写成主存大容量首选。 |
| DRAM | 需要刷新的动态随机存储器。 | 容量大、成本低，适合主存。 | 比较存储器时使用。 | 常考刷新原因。 | 不要忘记刷新。 |
| Cache | 位于 CPU 和主存之间的高速小容量存储。 | 利用局部性降低平均访存时间。 | 分析映射、替换、写策略和命中率。 | 常考 Cache 地址划分。 | 不要把 Cache 当成主存扩容。 |
| 虚拟存储 | 通过主存和外存配合提供更大的逻辑地址空间。 | 解决程序地址空间大于物理主存的问题。 | 按页号、页内偏移、页表转换分析。 | 常考地址转换。 | 不要混淆虚拟页号和物理页框号。 |

| 题型 | 题型特征 | 解题入口 | 解题步骤 | 作业书写模板 | 示例题 | 参考答案 | 易错点 |

| 概念辨析题 | 题干要求解释或区分概念 | 先找关键词，再回到概念关系 | 定义 -> 为什么引入 -> 与相近概念区别 -> 应用场景 | 本概念是……，它用于……，与……的区别是……。 | 示例：解释存储层次的含义。 | 参考：存储层次是由速度、容量和成本不同的存储部件构成的层级。引入它是因为单一存储器无法同时满足高速、大容量、低成本。 | 不要只按容量排序，忽略速度和成本。 |

| 综合题 | 题干跨多个概念或要求完整分析 | 先拆题，再分别调用对应知识点 | 拆分任务 -> 章节定位 -> 分步推理 -> 汇总结论 | 本题涉及……，先分析……，再处理……，最后得到……。 | 示例：围绕本章主题完成综合分析。 | 参考：分段作答并标明每段依据。 | 没有先分析题目结构。 |
| 实验或设计类题目 | 要求说明设计步骤、流程或报告内容 | 先明确目标和输入输出 | 目标 -> 方案 -> 步骤 -> 结果 -> 问题分析 | 本设计的目标是……，核心模块是……，验证方式是……。 | 示例：写出一个与本章相关的实验报告框架。 | 参考：写清目标、方法、结果和不足。 | 只给结果，不写设计理由。 |

## 12. 常见错误与纠正

- 错误：只背定义。纠正：同时写出为什么引入、解决什么问题、题目中怎么考。
- 错误：只给最终答案。纠正：保留题目分析、规则选择和关键过程。
- 错误：把相近概念混用。纠正：用比较维度区分定义、作用和适用场景。
- 错误：忽略 OCR 或图表不确定内容。纠正：在答案末尾标注“需人工复核”。
